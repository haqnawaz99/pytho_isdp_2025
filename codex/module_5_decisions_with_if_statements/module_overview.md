# Module 5 Overview: Decisions with If Statements

## Module title

Decisions with If Statements

## Recommended level

Beginner students who have completed Module 4

## Duration

6 lessons, 40 to 45 minutes each

## Module purpose

This module teaches students how a Python program can make decisions. Students learn how to run code only when a condition is true, how to choose between two or more options, and how to apply this to familiar real-life situations.

## Learning objectives

By the end of this module, students will be able to:

- write a simple `if` statement
- use `if-else` for two possible outcomes
- use `if-elif-else` for multiple outcomes
- compare user input in a condition
- perform simple validation checks
- write small decision-based programs using familiar contexts

## Prior knowledge

Students should be able to:

- use comparison operators
- understand `True` and `False`
- take user input
- convert input when needed

## Module progression

### Lesson 1

- Focus: basic `if`
- Outcome: students understand that code can run only when a condition is true

### Lesson 2

- Focus: `if-else`
- Outcome: students can write a two-way choice

### Lesson 3

- Focus: `if-elif-else`
- Outcome: students can handle three or more possible outcomes

### Lesson 4

- Focus: comparing user input
- Outcome: students can check typed answers with conditions

### Lesson 5

- Focus: simple validation
- Outcome: students can check whether input is acceptable

### Lesson 6

- Focus: real-life decision programs
- Outcome: students can create small useful programs with conditions

## Pakistan-relevant teaching contexts

- marks and grades
- weather and heat
- school attendance
- cricket score milestones
- age checks
- shopping decisions
- rainfall messages
- class or school reminders

## Teacher reminders

- Teach the structure visually on the board.
- Rehearse the difference between `=` and `==`.
- Spend time on indentation because this is a major beginner hurdle.
- Use real student predictions before running each example.
- Start with one condition only, then expand.

## Common misconceptions

- Students may use `=` instead of `==`.
- Students may forget the colon after `if`.
- Students may indent incorrectly.
- Students may expect every branch to run.

## End-of-module evidence

A student is ready for Module 6 if they can:

- write correct `if`, `if-else`, and `if-elif-else` structures
- compare a value and explain the result
- use input with a condition
- write a short decision-based program with clean indentation
