# Module 4 Overview: Operators and Calculations

## Module title

Operators and Calculations

## Recommended level

Beginner students who have completed Module 3

## Duration

6 lessons, 40 to 45 minutes each

## Module purpose

This module teaches students how Python performs calculations and comparisons. Students learn the meaning of common operators, use them in practical examples, and understand the order in which Python evaluates expressions.

## Learning objectives

By the end of this module, students will be able to:

- use arithmetic operators for simple calculations
- explain the difference between `/`, `//`, and `%`
- use `**` for powers
- write simple calculation-based programs
- compare values using comparison operators
- explain the meaning of `and`, `or`, and `not`
- use brackets to control operator precedence

## Prior knowledge

Students should be able to:

- work with integers and floats
- convert numeric input
- use variables and `f-strings`

## Module progression

### Lesson 1

- Focus: basic arithmetic operators
- Outcome: students can add, subtract, multiply, and divide

### Lesson 2

- Focus: floor division, remainder, and powers
- Outcome: students understand equal share, leftovers, and exponents

### Lesson 3

- Focus: practical calculations
- Outcome: students can write short real-life calculation programs

### Lesson 4

- Focus: comparison operators
- Outcome: students can compare values and read boolean results

### Lesson 5

- Focus: logical operators
- Outcome: students can combine true/false conditions conceptually

### Lesson 6

- Focus: operator precedence
- Outcome: students can predict calculation order and use brackets

## Pakistan-relevant teaching contexts

- rupee budgets
- canteen bills
- cricket scores
- marks and averages
- electricity units
- number of books
- equal distribution of notebooks
- rainfall and temperature

## Teacher reminders

- Move from very small number examples to real-life programs.
- Use the board to trace calculations step by step.
- Avoid mixing operators with `if` logic too early.
- Ask students to predict output before running code.
- Use practical examples like school fees, notebooks, and shopping.

## Common misconceptions

- Students may confuse `/` and `//`.
- Students may think `%` gives a percentage.
- Students may think `=` and `==` mean the same thing.
- Students may read `and` and `or` in everyday language, not as logic operators.

## End-of-module evidence

A student is ready for Module 5 if they can:

- use arithmetic operators accurately
- explain quotient and remainder
- compare values correctly
- interpret boolean results
- evaluate a short expression using precedence rules
