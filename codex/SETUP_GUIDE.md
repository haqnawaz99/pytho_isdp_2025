# Setup Guide

## Recommended environment

- Python 3.11 or newer
- Thonny for beginner-friendly classrooms
- IDLE as backup
- VS Code only if the lab already supports it well

## Minimum setup plan

1. Install Python
2. Confirm Python runs
3. Open the `codex` folder
4. Run one sample file from Module 1

## First test

Run a simple file such as:

- `module_1_printing_and_strings/code_examples/lesson_1_printing_messages.py`

Expected result:

- basic text should print on the screen without errors

## School lab advice

- Keep one backup USB copy of the course
- Use offline tools whenever possible
- Avoid relying on internet during lessons
- Test one machine fully before setting up the full lab

## Device-sharing advice

- Use pair programming
- Put one student on typing and one on checking logic
- Rotate roles every 10 to 15 minutes

## Common setup problems

- Python not installed
- file opened outside the correct folder
- students running code from the wrong file
- keyboard typing issues, especially brackets and quotes
