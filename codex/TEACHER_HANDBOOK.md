# Teacher Handbook

## Teaching priorities

1. Correct understanding before speed
2. One new concept at a time
3. Frequent prediction before running code
4. Local, familiar examples
5. Short revision every lesson

## Recommended lesson flow

1. 5-minute recap
2. today's objective
3. teacher demo
4. guided practice
5. independent practice
6. mistake review
7. exit ticket

## Common beginner trouble points

### Module 1

- quotes
- spaces in concatenation
- difference between code and output

### Module 2

- variable names
- difference between variable and string
- understanding `input()`

### Module 3

- `"25"` vs `25`
- `input()` returns text
- wrong conversion choice

### Module 4

- `/` vs `//`
- `%` as remainder, not percentage
- `=` vs `==`

### Module 5

- missing colon
- indentation
- wrong branch order

### Module 6

- `range()` confusion
- infinite `while` loops
- updating counters

### Module 7

- defining vs calling
- printing vs returning
- forgetting arguments

### Module 8

- indexing starts at 0
- lists vs sets vs dictionaries
- keys are not indexes

### Module 9

- building too much at once
- not testing enough
- weak variable names

## Language guidance

- Teach code in English only.
- Explain ideas in simple English plus Urdu support where needed.
- Keep technical keywords consistent.

## Classroom management in limited-resource settings

- use pair programming
- prefer short runnable examples
- use whiteboard tracing often
- let one student explain code to another

## Good teaching rule

If students cannot explain what the code is doing, they are not ready to move on.
