# Module 9 Overview: Mini Projects

## Module title

Mini Projects

## Recommended level

Students who have completed Modules 1 to 8

## Duration

6 lessons, 45 to 60 minutes each

## Module purpose

This module gives students a chance to combine the full beginner Python course into practical projects. Students review earlier concepts, build guided projects, and present or explain their work.

## Learning objectives

By the end of this module, students will be able to:

- combine multiple Python concepts in one program
- plan a small project before writing code
- write readable beginner-level project code
- test a project with sample inputs
- explain how their program works
- improve a project after feedback

## Prior knowledge

Students should be able to:

- use variables, input, and output
- work with numbers and conversion
- use conditions and loops
- define and call functions
- use lists or dictionaries at a simple level

## Module progression

### Lesson 1

- Focus: project skills review
- Outcome: students identify which earlier topics they need

### Lesson 2

- Focus: school canteen bill project
- Outcome: students build a calculation-based project

### Lesson 3

- Focus: student report card project
- Outcome: students build a condition-based and function-based project

### Lesson 4

- Focus: cricket score helper project
- Outcome: students build a sports-related logic project

### Lesson 5

- Focus: weather and rainfall helper project
- Outcome: students build a local-context advisory project

### Lesson 6

- Focus: showcase and reflection
- Outcome: students present, improve, and reflect on project work

## Key vocabulary

## Pakistan-relevant project contexts

- school canteen bills
- report cards and marks
- cricket scoring
- weather and rainfall messages
- shopping and family budgets
- study planners

## Teacher reminders

- Keep projects short enough to finish.
- Use guided structure for weaker students.
- Encourage pair programming where devices are limited.
- Assess clarity, logic, and explanation, not only final output.
- Let students personalize names, cities, and examples.

## Common misconceptions

- Students may try to build too much at once.
- Students may forget to test with different inputs.
- Students may copy code without understanding each part.
- Students may think a project is only about length, not quality.

## End-of-module evidence

A student has completed the beginner course successfully if they can:

- build at least one complete working project
- explain the purpose of each major code section
- test the project with more than one input
- improve the code after teacher feedback
