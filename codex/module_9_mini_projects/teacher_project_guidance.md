# Teacher Project Guidance

## Recommended teaching approach

1. Start with one whole-class guided project.
2. Move to paired or individual project work.
3. Let students reuse earlier module examples when useful.
4. Keep projects short enough to complete and improve.

## Suggested order

1. School canteen bill
2. Student report card
3. Cricket score helper
4. Weather and rainfall helper

## Differentiation

### For struggling students

- give a starter template
- reduce the number of inputs
- allow pair programming
- focus on one working feature first

### For stronger students

- add validation
- add one extra feature
- use one function more than required
- test with more sample inputs

## Common teacher moves

- Ask students to say the plan before they code.
- Ask what each input is for.
- Ask what should happen if the value is unusual.
- Ask whether the code is readable to someone else.

## Good project expectations

- clear purpose
- correct input handling
- correct calculation or decision logic
- understandable output
- ability to explain code choices
