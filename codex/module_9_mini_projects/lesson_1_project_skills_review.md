# Lesson 1: Project Skills Review

## Duration

45 to 60 minutes

## Learning objective

Students will be able to identify the Python skills needed for a small project and review them before coding.

## Key vocabulary

- project
- plan
- review

## Starter

Ask students:

- What skills do we need to build a useful Python program?

## Concept explanation

Before writing a project, students should review:

- input and output
- numbers and conversion
- conditions
- loops
- functions
- lists or dictionaries

Good projects start with a simple plan:

1. What problem will the project solve?
2. What inputs are needed?
3. What output should the program show?
4. Which Python ideas will be used?

## Worked example 1

Project plan for a canteen bill:

- input: item price, quantity
- process: calculate total
- output: total bill
- concepts: variables, numbers, operators, functions

## Worked example 2

Project plan for a report card:

- input: student name and marks
- process: calculate result category
- output: report
- concepts: input, conversion, `if`, functions

## Worked example 3

Project plan for a weather helper:

- input: temperature and rainfall
- process: classify conditions
- output: advice
- concepts: input, float conversion, `if-elif-else`

## Guided practice

Students choose one project and fill in:

- project name
- input
- process
- output

## Independent practice

### Beginner

Plan a canteen bill project.

### Intermediate

Plan a cricket score helper.

### Challenge

Plan your own simple school or home project.

## Common mistakes

- starting coding before planning
- choosing too many features at once
- not knowing which earlier skills are needed

## Assessment checkpoint

1. Why do we plan before coding?
2. What is the difference between input and output?
3. Why should projects stay small at this level?

## Exit ticket

Students write a 4-line project plan.

## Homework

Choose one project idea and list the Python concepts it will use.
