# Lesson 6: Project Showcase and Reflection

## Duration

45 to 60 minutes

## Learning objective

Students will be able to present their mini project, explain their code, and improve it after feedback.

## Key vocabulary

- present
- explain
- feedback
- improve

## Starter

Ask students:

- What makes a project successful besides “it runs”?

## Concept explanation

Good project work includes:

- clear purpose
- correct logic
- readable variable names
- tested inputs
- ability to explain the code

## Showcase checklist

Students should explain:

1. project name
2. what problem it solves
3. what inputs it takes
4. what output it gives
5. which Python ideas it uses

## Reflection prompts

Students write or discuss:

1. What part of the project was easiest?
2. What part was hardest?
3. What bug did you fix?
4. What would you improve next?

## Guided practice

Students test each other's projects using sample inputs.

## Independent practice

### Beginner

Explain one part of your project to a partner.

### Intermediate

Find and fix one issue in your own project.

### Challenge

Improve your project after feedback and explain the change.

## Common mistakes

- only demonstrating one input
- not checking unexpected inputs
- describing output without explaining logic

## Assessment checkpoint

1. Why should we test with more than one input?
2. Why do clear variable names matter?
3. What is one useful way to improve a project?

## Exit ticket

Students write one sentence about what they learned from building a project.

## Homework

Submit the final improved version of one project with a short written explanation.
