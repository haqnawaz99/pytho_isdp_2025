# Project Rubric

## Scoring scale

- 1 = Beginning
- 2 = Developing
- 3 = Secure
- 4 = Strong

## Rubric areas

### 1. Problem understanding

- 1: The project goal is unclear.
- 2: The goal is partly clear.
- 3: The goal is clear and appropriate.
- 4: The goal is clear, useful, and well explained.

### 2. Correct use of Python concepts

- 1: Major errors stop the program.
- 2: Some concepts are correct, but there are logic gaps.
- 3: Most concepts are used correctly.
- 4: Concepts are used correctly and confidently.

### 3. Code clarity

- 1: Variable names and structure are unclear.
- 2: Some parts are clear.
- 3: Code is mostly readable and organized.
- 4: Code is clear, consistent, and easy to explain.

### 4. Testing

- 1: Project was not properly tested.
- 2: Only one test was shown.
- 3: More than one test was shown.
- 4: Multiple useful tests were shown, including edge cases.

### 5. Explanation

- 1: Student cannot explain the program clearly.
- 2: Student explains some parts.
- 3: Student explains the main logic.
- 4: Student explains the logic clearly and confidently.
