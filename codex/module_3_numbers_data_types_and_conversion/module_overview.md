# Module 3 Overview: Numbers, Data Types, and Type Conversion

## Module title

Numbers, Data Types, and Type Conversion

## Recommended level

Beginner students who have completed Module 2

## Duration

6 lessons, 40 to 45 minutes each

## Module purpose

This module helps students understand that Python stores different kinds of data in different forms. Students learn the difference between text and numbers, how to check a value's type, and how to convert input into the correct form for later use.

## Learning objectives

By the end of this module, students will be able to:

- identify integers and floating-point numbers
- explain the difference between `str`, `int`, `float`, and `bool`
- use `type()` to inspect a value
- convert text to numbers using `int()` and `float()`
- convert numbers to text using `str()`
- explain why `input()` often needs conversion before calculation
- recognize and avoid common conversion errors

## Prior knowledge

Students should be able to:

- create variables
- use `input()`
- use variables in output
- read a simple `f-string`

## Module progression

### Lesson 1

- Focus: numbers in Python
- Outcome: students can identify whole numbers and decimal numbers

### Lesson 2

- Focus: basic data types
- Outcome: students can describe `str`, `int`, `float`, and `bool`

### Lesson 3

- Focus: `type()`
- Outcome: students can check what type a value has

### Lesson 4

- Focus: type conversion
- Outcome: students can convert between string and number forms

### Lesson 5

- Focus: converting user input
- Outcome: students can convert typed input before using it

### Lesson 6

- Focus: common conversion mistakes
- Outcome: students can identify and fix beginner conversion errors

## Teacher reminders

- Keep examples small and concrete.
- Use `type()` often to make the invisible visible.
- Show the difference between `"25"` and `25` repeatedly.
- Avoid jumping to full arithmetic drills; that belongs mainly in the next module.
- Spend time on error understanding, not only correct answers.

## Common misconceptions

- Students may think `"25"` and `25` are the same thing.
- Students may think `input()` gives an integer automatically.
- Students may think `float` only means difficult maths.
- Students may confuse value with type.

## End-of-module evidence

A student is ready for Module 4 if they can:

- identify whether a value is `str`, `int`, `float`, or `bool`
- use `type()` correctly
- convert a string number into `int` or `float`
- explain why `str()` is needed in some printed messages
- fix a simple conversion mistake
