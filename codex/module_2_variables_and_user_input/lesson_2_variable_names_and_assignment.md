# Lesson 2: Variable Names and Assignment

## Duration

45 minutes

## Learning objective

Students will be able to assign values to variables and choose valid, meaningful variable names.

## Key vocabulary

- assignment
- variable name
- meaningful name

## Starter

Show students these names:

- `name`
- `student_name`
- `my city`
- `2age`

Ask:

- Which ones look easier to understand?
- Which ones do you think Python will reject?

## Concept explanation

In Python, the equals sign `=` is used for assignment. It stores a value in a variable.

Good variable names:

- should describe the value
- should not contain spaces
- should not start with a number

## Worked example 1

```python
student_name = "Ayesha"
print(student_name)
```

## Worked example 2

```python
favorite_subject = "Science"
print(favorite_subject)
```

## Worked example 3

```python
madrasa_name = "Jamia Noor"
print(madrasa_name)
```

## Guided practice

Tell students to fix bad names:

- `my name`
- `1city`
- `favorite-food`

Possible corrected forms:

- `my_name`
- `city1`
- `favorite_food`

## Independent practice

### Beginner

Create a variable called `fruit` and store `Mango`.

### Intermediate

Create a variable called `school_name` and print it.

### Challenge

Create three meaningful variable names for name, class, and favorite game.

## Common mistakes

- using spaces in names
- starting names with numbers
- using unclear names like `x` for beginner activities

## Assessment checkpoint

1. What does `=` do in Python?
2. Why is `student_name` better than `x` for beginners?
3. Is `my city` a valid variable name?

## Exit ticket

Students write one valid variable name and one invalid variable name.

## Homework

Write five good variable names for things from school or daily life.
