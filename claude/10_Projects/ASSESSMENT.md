# Module 10 — Projects: Assessment

## Assessment Approach

Assessment for this module is **practical and observational**. The teacher runs each student's program live and observes whether it works correctly. There is no written quiz — the code itself is the evidence.

> Run the program. Give it real inputs. Try to break it with bad inputs. Watch what happens.

---

## Pass Criteria — Per Project

### Project 1: Report Card System

A student passes this project when their program can:

- [ ] Add a student with name, subject marks, and calculate average
- [ ] View all students and their averages
- [ ] Search for a student by name and display their record
- [ ] Show a class summary (highest, lowest, class average)

### Project 2: Prayer Reminder

A student passes this project when their program can:

- [ ] Display the 5 daily prayer schedule with times
- [ ] Mark a prayer as prayed or missed for today
- [ ] View the prayer history from file
- [ ] Show correct statistics (e.g. percentage of prayers completed)

### Project 3: Quiz Game

A student passes this project when their program can:

- [ ] Play a quiz with at least one topic
- [ ] Validate the answer (only accept a/b/c/d)
- [ ] Calculate and display the score at the end
- [ ] Save the score to the leaderboard file
- [ ] Display the leaderboard with past scores

## Portfolio Submission

Each student submits the following:

1. `10_1_report_card_system.py` — their completed file
2. `10_2_prayer_reminder.py` — their completed file
3. `10_3_quiz_game.py` — their completed file
4. A screenshot of **each program running** (3 screenshots total)

> Screenshots can be taken with the Snipping Tool (Win + Shift + S) and saved to the student's folder.

## Notes for Teacher

- If a student's program crashes, ask them to **explain what the code should do** — understanding without working code earns partial marks.
- Students who used the starter file without modification score 0 — they must have made meaningful additions.
- Celebrate finishing! Completing 3 real programs is a significant achievement. Make dua together and acknowledge the effort.
- Consider: for YouTube content, record one strong student's project running as a demonstration.
