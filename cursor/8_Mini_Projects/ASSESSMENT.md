# Module 8 — Assessment & Course Completion

## Per-project rubric (each out of 10)

| Criteria | 0–2 | 3–5 | 6–8 | 9–10 |
|----------|-----|-----|-----|------|
| Runs without error | Crashes | Minor fix needed | Runs | Runs all inputs |
| Correct logic | Wrong | Partly correct | Mostly correct | Fully correct |
| Own context | Copy only | One change | Several changes | Clear personalization |
| Code style | Messy | Some names | Clear names, indent | + comments |

**Pass per project:** 6 or above  
**Course pass projects:** At least **3 of 5** projects at pass level, including **project 5** OR **project 3** for struggling students (teacher discretion).

## Final written quiz (optional, 10 MCQ sample)

1. Which module taught `if` / `else`? → Module 5  
2. `student["name"]` is used when data is in a → dictionary  
3. `for x in items:` needs `items` to be a → list (or string)  
4. `def` is used to create a → function  
5. `int(input())` is needed because input returns → str
