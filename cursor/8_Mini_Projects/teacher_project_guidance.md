# Teacher Project Guidance (Module 8)

Adopted from **codex** `teacher_project_guidance.md`.

## Recommended approach

1. **Week 1:** Whole-class guided — complete `8_1_attendance` together from starter.  
2. **Week 2–3:** Pairs choose 2 more projects from starters.  
3. **Week 4:** Integrated `8_5_class_report` for students who finished early.  
4. **Showcase day:** `SHOWCASE_AND_REFLECTION.md`.

## Suggested project order

| Order | Project | Why |
|-------|---------|-----|
| 1 | Attendance `8_1` | Easiest; input + if |
| 2 | Maktaba `8_2` or Name `8_4` | Lists or strings |
| 3 | Fee `8_3` | Numbers + optional function |
| 4 | Class report `8_5` | Capstone |
| 5 | Cricket `8_6` or Weather `8_7` | Optional motivation |

## Differentiation

### Struggling students

- Use starter only; teacher fills one TODO live.  
- Reduce to **2 projects** (8_1 + one choice).  
- Pair with stronger student for typing only.  

### Stronger students

- Add input validation (`isdigit()`).  
- Add a function per project.  
- Complete `8_5` + one optional project.  
- Help peer review using `project_rubric.md`.

## Teacher questions (during lab)

- What is the **input**?  
- What is the **output**?  
- What happens if user types something unexpected?  
- Point to one `if` and one loop in your code.  

## Do not require in Year 1

- File saving (`open`) — see `9_Advanced_Optional`  
- `try/except` — second term  
- Copy-paste from `8_*_solution.py` without explanation — ask them to explain one line
