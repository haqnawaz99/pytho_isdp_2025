# Module 0 — Teacher Notes

## Time guide (flexible)

| Lesson | Suggested time | Notes |
|--------|----------------|-------|
| 0.1 What is programming? | 20–30 min | Unplugged: student writes “algorithm” for making chai on paper |
| 0.2 VS Code setup | 30–45 min | Do once in lab; pair strong + weak student |
| 0.3 `print()` | 40–50 min | Most hands-on keyboard time |
| 0.4 Comments & errors | 30–40 min | Deliberately break code together |

## Delivery tips

1. **Do not rush typing.** First lesson anxiety is normal. Walk the room.
2. **One concept per file.** Use `0_3_first_print.py` only for print; errors go in `0_4`.
3. **Mixed audience:** School students may type faster; pair them with madrasa students for setup only, not for grading competition.
4. **Islamic examples:** Use naturally, not as a test of religious practice (e.g. do not ask students to prove they prayed).

## Common misconceptions

| Student says | Truth |
|--------------|-------|
| “Computer understands Urdu in print” | Only the **text in quotes** is shown; Python keywords are English |
| “Comment runs slowly” | Comments do not run at all |
| “Error means program is deleted” | File is safe; fix and run again |

## Extension (fast finishers)

- Print ASCII art with multiple `print()` lines (mosque outline, simple flag).
- `print("Line 1", "Line 2")` — comma adds space between items (preview of later).

## Homework (low resource)

On paper: write 5-step “program” for opening a classroom (open door, switch light, etc.). Next class, one student reads steps — link to sequential execution.
